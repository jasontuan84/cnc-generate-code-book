Chapter 4 · Outline

Outline & Contour

The outline seems like the simplest chain source of all: just trace the shape's boundary. But "trace the boundary" hides a few decisions that are anything but trivial - when it should actually be emitted, how it interacts with fills, and the physical compensations a cutting blade needs (overcut, kerf/tool offset). All of this is independent of the target machine language.

1. What an outline is

The outline chains of a shape are the polylines of its boundary path after the shape has gone through three processing steps:

1 · Bake transforms into world space
→
2 · Flatten curves at the fine tolerance
→
3 · Split into subpaths (outer boundary + each hole)

Specifically: (1) every transform (translate/rotate/scale/shear) is baked into world space (see Ch. 1 §2); (2) curves are flattened at the fine tolerance (Ch. 2 §1); (3) the path is split into its subpaths - the outer boundary plus each hole boundary - each subpath becoming one chain.

Rule: subpath → chain

Every subpath with 2 or more points becomes exactly one chain. A closed subpath becomes a closed chain (first point ≈ last point); an open subpath - an unclosed stroke, or a path imported from a file that was never closed - stays an open chain. Nothing else is required: an outline is purely the geometry of the boundary, independent of any fill rule or winding direction.

2. When outlines are emitted (the presence rules)

An outline is emitted purely from the shape's geometry/style - independent of the machine mode - when either of the following holds:

ConditionReason
Shape has a stroke styleThere is an explicit border line that needs to be drawn.
Shape has no fillWith nothing to fill, the outline is the artwork.

Conversely, a shape that is filled but has no stroke emits only its fill chains (Ch. 3), not an outline - the design specifies no border, so none is invented.

Don't rely on "the fill already reaches the boundary"

It's true that a scanline fill does reach all the way to the boundary - but inset-based fills (contour/spiral, Ch. 3 §3.4) stop about half a line spacing short of the boundary and leave a thin unfilled rim. If exact edge coverage matters on a borderless filled shape, make the fill hug the boundary more tightly rather than bolting on a stray outline.

2b. Shapes with both fill and stroke - emission decisions

When a shape has both a fill and a stroke, it produces two chain sets over the same boundary region. The additive dispatch rule says "emit both" - but leaves three concrete decisions that the theory alone doesn't pin down:

Settings
A shape's fill and outline live on the same layer/tool-group, so they inherit the same power/speed/pass-count (Ch. 8 §7). If the outline needs different settings (darker, or cut all the way through, while the fill is only lightly marked), put it on a separate layer.
Order
Prefer fill first, then outline, so a crisp border isn't smeared by later fill passes. Greedy ordering (Ch. 7) does not guarantee this on its own - use an affinity/grouping key (Ch. 7 §5) to keep a shape's fill-then-outline pair travelling together.
Shared edge / double pass
The outermost lines of a scanline fill and the outline both sit close to the boundary; on a marking tool, this shared edge may get passed over twice. Usually harmless (even beneficial - a crisper edge). To avoid it, inset the fill by ½·lineWidth so the outline is the sole "owner" of the boundary. Note this is not the endpoint welding of Ch. 10 §3.1: welding joins fragments within the same chain source; it never merges a fill with a separate outline.

3. Outline vs Contour fill: don't confuse them

They share the same primitive - trace the boundary - but the two features answer two different questions, and must be kept distinct:

Outline (this chapter)

The boundary drawn once. A single pass along each subpath. Answers "draw the boundary".

Contour fill (Ch. 3 §3.4)

The boundary drawn many times, each inset inward by one line spacing, to cover the interior area. Answers "fill the interior by echoing the boundary".

4. Optional tool compensations (blade/knife/kerf)

These are optional post-processing steps applied to already-chosen outline chains, relevant only when the tool is a physical cutting blade. A marking tool (pen, laser marking) needs none of these - the mark sits directly on the path. They are not a separate mode or pipeline; for a marking tool, skip them entirely.

Order of operations relative to ordering (Ch. 7)

Apply kerf offset BEFORE ordering - it changes the geometry that the ordering and loop re-rooting steps will operate on. Apply overcut AFTER ordering and loop re-rooting - overcut extends past the loop's final start vertex, so it must run on the oriented, re-rooted chain, re-crossing wherever the seam ended up. Reversing this order (overcut before re-rooting) places the extension mid-loop and fails to sever anything.

4.1 · Overcut
Extend a small distance past the start point of a closed loop so the blade fully severs the seam.
4.2 · Kerf / tool offset
Offset the path by half the kerf width so the cut size matches the drawn size.
4.3 · Corner sharpening
Compensate for corners rounded off because the blade trails the pivot.

4.1 Overcut (fully closing a severed loop)

A drag knife (used in vinyl/decal cutters) has its blade tip positioned behind its pivot axis. On a closed loop, the very start of the loop is only scored until the blade has rotated to face the next direction; the cut loop may not fully separate at the seam. Overcut fixes this:

Overcut definition

After closing a loop, extend the path a small distance (≈ 0.5-1.0 mm) past the start point, continuing in the direction of the first segment, so the blade re-crosses the seam and fully severs it.

The algorithm, applied after a chain has been closed into a loop:

if chain.is_closed and overcut > 0:
    dir = unit(chain[1] − chain[0])
    chain.append(chain[0] + dir × overcut)  // extend past the start point

Apply only to closed chains and only when the tool is a cutting tool. For a marking tool, this extension would leave a stray tick mark - do not apply. It is a no-op for open chains or a negligible overcut value.

4.2 Kerf / tool offset

The tool removes (or marks) a certain material width. To cut the exact size drawn, offset the path by half the kerf width:

  • Offset outward when cutting a hole to size, or an outer boundary that must keep its outer dimension.
  • Offset inward for the opposite intent.

Uses the same polygon offsetting primitive as contour fill (Ch. 2 §7). Many simple cutters skip true kerf compensation and instead record the installed blade offset as a calibration note, sharpening corners with a small extra travel move; full kerf compensation is the general solution when dimensional accuracy really matters.

Offset direction is the easiest thing to get wrong

Offsetting a boundary in the wrong direction changes the part's dimension by a full kerf width. Tie the offset direction to whether each subpath is an outer boundary or a hole (winding/even-odd depth, Ch. 2 §4).

4.3 Corner sharpening (drag knife)

Because the blade trails the pivot, sharp corners get rounded off. Standard fix: at each corner, lift-and-swivel or add extra travel equal to the blade-offset length so the blade tip reaches the true corner before proceeding onto the next edge. This is a per-corner refinement of the same idea behind overcut.

5. Multi-pass

Thick material (or a need for a darker mark) requires several passes over the same outline. Emit the ordered chains, then repeat that entire set N times (usually lowering the tool slightly each pass, if the machine has a Z axis).

Order once, repeat the result

Order the chains exactly once; repeat that ordered result so every pass follows the same travel-optimised path. Do not re-order per pass.

6. Stroke variants: dashed & variable-width

Two common stroke styles need pre-processing before the boundary becomes chains:

6.1 Dashed / dotted strokes

A dash array turns a continuous boundary into many short open chains with gaps between them. Walk along the path by arc length, emitting an "on" segment for each dash length and skipping each gap length. Each dash is an independent chain (the ordering step will travel between them). Do this after flattening (measuring length along the polyline).

6.2 Variable-width strokes (calligraphic, pressure, tapered)

Two choices:

  • (a) treat the stroke as a filled region: expand the centerline into its variable-width outline and fill it (Ch. 3); or
  • (b) if the tool's mark width already matches, just draw the centerline once and ignore the width.

Choose (a) when the width variation is visible at tool scale; choose (b) when it isn't.

7. Summary: open & closed handling

CaseOutline behaviour
Closed subpath, marking toolTrace once; no overcut.
Closed subpath, cutting toolTrace once + overcut; optional kerf offset.
Open subpathTrace from one endpoint to the other; ordering may reverse it; never overcut.
Hole subpathEmitted like any other subpath; for a cutting tool, offset in the opposite direction to the outer boundary.
"Marking tool" vs "cutting tool"

This is a distinction of tool capability (does the tool sever material?), not a UI/job mode. It only changes whether the optional compensations in §4 apply; the outline chains themselves are produced identically in both cases.

8. Checklist (when generating outline chains)

  • Bake every transform, then flatten at the fine tolerance.
  • Emit one chain per subpath (≥ 2 points).
  • Apply the presence rules (§2) so outline and fill don't unintentionally overlap.
  • Cutting tool only: overcut closed loops; consider kerf offset with the correct direction per subpath role.
  • Repeat the ordered chains for multi-pass; don't re-order per pass.
  • Return chains in world space; ordering and code emission are shared across all chain sources.
