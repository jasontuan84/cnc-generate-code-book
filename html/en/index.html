Technical book

From Vector to Machine Code

An A-to-Z journey turning a 2-D vector drawing into the motion commands a laser, vinyl cutter, engraver, or pen plotter can execute - explaining the decisions, logic and algorithms, independent of any single machine model or programming language.

Version 1.0.0 · Updated 2026-07-02 · English edition

Who is this book for?

For developers who want to build their own code generator (CAM) for a point-to-point machine, for engineers who want to understand why their laser/cutting software behaves the way it does, and for AI assistants tasked with generating this kind of code. You do not need prior knowledge of G-code or HPGL; you only need to be comfortable with basic 2-D geometry and a little algorithmic thinking.

The guiding philosophy

This is a theory-first, machine-agnostic reference. Every technique named here is a standard, named algorithm you can implement in any language. Each concrete number (a threshold, a tolerance) is a tuned default with a stated rationale - not a magic constant.

Reading path

You can read straight through, but here is the recommended path for someone new to building a code generator:

Introduction→ Ch. 1 Architecture→ Ch. 2 Geometry→ Ch. 3-5 Chain sources→ Ch. 7 Ordering→ Ch. 8 Emission

Chapters 6 (pattern) and 9 (raster) are read when those features are needed. Chapters 10 (fidelity) and 11 (performance) are cross-cutting - apply them to every stage as you build. Read chapter 12 (edge cases) before shipping - it is the hardening checklist.
